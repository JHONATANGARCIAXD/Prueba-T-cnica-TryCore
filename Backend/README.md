## ⚙️ Configuración del Entorno

### 1. Configurar tu Proyecto en Supabase

Si aún no tienes un proyecto en Supabase:

1. **Ve a [supabase.com](https://supabase.com)** y crea una cuenta gratuita
2. **Crea un nuevo proyecto**:
   - Nombre: Elige un nombre (ej: "trycore")
   - Password: Establece una contraseña segura (guárdala, la necesitarás)
   - Region: Elige la más cercana a tu ubicación
3. **Espera a que se inicialice** (2-3 minutos)

### 2. Obtener la URL de Conexión

Una vez que el proyecto esté listo:

1. **Ve a Project Settings** (icono inferior izquierda)
2. **Selecciona Database**
3. **Busca "Connection string"**
4. **Copia la URL**:
   ```
   postgresql://postgres:<YOUR_PASSWORD>@db.supabase.co:5432/postgres
   ```

### 3. Crear las Tablas

1. **Ve al Editor SQL** en Supabase (opción en el menu izquierdo)
2. **Haz clic en "+ New Query"**
3. **Copia y pega este código SQL**:

```sql
-- Crear tabla de productos
CREATE TABLE public.product (
  id bigint generated by default as identity not null,
  name text null,
  price integer null,
  current_stock integer null,
  min_stock integer null,
  created_at timestamp with time zone not null default now(),
  constraint product_pkey primary key (id)
) TABLESPACE pg_default;

-- Crear tabla de movimientos
CREATE TABLE public.movement (
  id bigint generated by default as identity not null,
  product_id bigint null,
  type text null,
  quantity integer null,
  created_at timestamp with time zone not null default now(),
  constraint movement_pkey primary key (id),
  constraint movement_product_id_fkey foreign key (product_id) references product (id)
) TABLESPACE pg_default;
```

4. **Haz clic en "Run"** o presiona `Ctrl+Enter`
5. ✅ **Listo**, las tablas están creadas

### 4. Crear el archivo `.env`

En la raíz del directorio `Backend/`, crea un archivo `.env` con las siguientes variables:

```env
# Puerto donde correrá el servidor
PORT=3000

# Configuración de la base de datos Supabase
# URL que copiaste en el Paso 2
DATABASE_URL=postgresql://postgres:tu_password@db.supabase.co:5432/postgres

```

**Variables de conexión:**

| Variable | Descripción | Ejemplo |
|----------|-------------|---------|
| `PORT` | Puerto del servidor Express | `3000` |
| `DATABASE_URL` | URL de conexión a Supabase | `postgresql://postgres:xxxx@db.supabase.co:5432/postgres` |

## 🚀 Pasos para Levantar el Backend

### 1. Navegar al directorio del Backend

```bash
cd Backend
```

### 2. Instalar dependencias

```bash
npm install
```

Este comando instalará:
- **express**: Framework web
- **cors**: Manejo de CORS
- **dotenv**: Variables de entorno
- **express-validator**: Validación de campos
- **pg**: Driver PostgreSQL (para conectar con Supabase)
- **nodemon**: (dev) Recarga automática en cambios

### 3. Verificar la conexión a la base de datos

Asegúrate de que:
- El archivo `.env` tiene la URL de Supabase correcta
- Copiaste exactamente la URL de conexión (incluyendo contraseña)
- Las tablas fueron creadas en Supabase (Paso 3 anterior)

### 4. Iniciar el servidor

```bash
npm run dev
```

**Salida esperada:**
```
Database Connected
Server is running on port 3000
```

El servidor estará disponible en: `http://localhost:3000`

### 5. Probar los endpoints

El servidor expone los siguientes endpoints base:

- **Productos**: `http://localhost:3000/api/products`
- **Movimientos**: `http://localhost:3000/api/movements`

## ❌ Problemas Comunes y Soluciones

### 1. Error: "Database connection error"

**Síntomas:**
```
Database connection error
```

**Soluciones:**
- [ ] Verifica que hayas copiado correctamente la URL de Supabase en `.env`
  - Debe empezar con: `postgresql://postgres:...`
- [ ] Asegúrate de no haber modificado la URL de conexión
- [ ] Comprueba que tu proyecto Supabase está activo (ve a supabase.com)
- [ ] Si todo parece correcto, regenera la contraseña en Supabase:
  - Ve a: **Project Settings** → **Database** → **Reset database password**
  - Copia nuevamente la URL de conexión
  - Actualiza `.env` con la nueva URL

### 2. Error: "Port 3000 is already in use"

**Soluciones:**
- Cambia el puerto en `.env`:
  ```env
  PORT=3001
  ```

## 📝 Notas Importantes


- **Supabase es la nube**: No necesitas instalar nada localmente para la BD
- **URL de conexión**: Asegúrate de tener la URL correcta en `.env` en todo momento
- **Puerto por defecto**: El servidor corre en puerto `3000`. Puedes cambiarlo en `.env`
- **CORS habilitado**: El servidor acepta solicitudes desde cualquier origen
