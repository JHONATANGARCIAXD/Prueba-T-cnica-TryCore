## 🛠️ Stack Tecnológico

### Backend
- **Lenguaje**: JavaScript (Node.js)
- **Framework**: Express 5.2.1
- **Base de Datos**: Supabase (PostgreSQL en la nube)
- **ORM/Driver**: pg (node-postgres)
- **Validación**: express-validator
- **Middleware**: CORS, dotenv
- **Dev Server**: Nodemon

### Frontend
- **Lenguaje**: JavaScript (ES6+)
- **Framework**: Vue 3
- **Framework de Componentes**: Quasar 2
- **Build Tool**: Vite
- **Enrutamiento**: Vue Router 4
- **Cliente HTTP**: Axios
- **Pre-procesador**: SASS

### Base de Datos
- **Servicio**: Supabase (PostgreSQL en la nube)
- **Tabla 1**: `product` (productos con stock)
- **Tabla 2**: `movement` (movimientos de inventario)

## 🗄️ Levantar Backend

### Paso 1: Configurar Supabase

Si aún no tienes un proyecto en Supabase:

1. **Ve a [supabase.com](https://supabase.com)** y crea una cuenta
2. **Crea un nuevo proyecto** (elige la región más cercana)
3. **Espera a que se inicialice** (suele tardar 2-3 minutos)
4. **Obtén la URL de conexión**:
   - Ve a: **Project Settings** → **Database**
   - Copia la **Connection String** bajo "URI"
   - Debería verse como: `postgresql://postgres:<password>@<host>:<port>/postgres`

### Paso 2: Crear las Tablas en Supabase

1. **Ve al editor SQL** en Supabase (opción de la izquierda)
2. **Haz clic en "New Query"**
3. **Copia y pega este SQL**:

```sql
-- Crear tabla de productos
CREATE TABLE public.product (
  id bigint generated by default as identity not null,
  name text null,
  price integer null,
  current_stock integer null,
  min_stock integer null,
  created_at timestamp with time zone not null default now(),
  constraint product_pkey primary key (id)
) TABLESPACE pg_default;

-- Crear tabla de movimientos
CREATE TABLE public.movement (
  id bigint generated by default as identity not null,
  product_id bigint null,
  type text null,
  quantity integer null,
  created_at timestamp with time zone not null default now(),
  constraint movement_pkey primary key (id),
  constraint movement_product_id_fkey foreign key (product_id) references product (id)
) TABLESPACE pg_default;
```

4. **Haz clic en "Run"** o presiona `Ctrl+Enter`
5. **Listo**, las tablas están creadas

### Paso 3: Configurar Variables de Entorno

En `Backend/`, crea un archivo llamado `.env`:

```bash
# Windows (en PowerShell o CMD)
cd Backend
New-Item -Name ".env" -ItemType "file"

# macOS/Linux
cd Backend
touch .env
```

Abre el archivo `.env` con un editor de texto y añade:

```env
PORT=3000
DATABASE_URL=<PEGA_AQUI_LA_URL_DE_SUPABASE>
NODE_ENV=development
```

**Reemplaza `<PEGA_AQUI_LA_URL_DE_SUPABASE>` con la URL que copiaste de Supabase** en el Paso 1.

Debería verse así (ejemplo):
```env
PORT=3000
DATABASE_URL=postgresql://postgres:abc123xyz@db.supabase.co:5432/postgres
NODE_ENV=development
```

### Paso 4: Instalar Dependencias del Backend

```bash
# Asegúrate de estar en Backend/
cd Backend

# Instala dependencias
npm install
```

Espera a que termine la instalación. Verás un mensaje similar a:

```
added XX packages in Xs
```

### Paso 5: Iniciar el Servidor Backend

```bash
npm run dev
```

**Resultado esperado:**
```
Database Connected
Server is running on port 3000
```

✅ **Backend levantado correctamente** en `http://localhost:3000`

## ❌ Problemas Comunes

### 1. "Database connection error"

**Causa**: La URL de Supabase es incorrecta o faltan credenciales.

**Soluciones**:
- [ ] Verifica que copiaste correctamente la URL de Supabase
  - Debe estar en `.env` como: `DATABASE_URL=postgresql://...`
- [ ] Asegúrate de no cambiar ni alterar la URL
- [ ] Comprueba que tu proyecto Supabase está activo
- [ ] Si el error persiste, regenera la contraseña en Supabase:
  - Ve a: **Project Settings** → **Database** → **Reset database password**

### 2. "Port 3000 is already in use"

**Causa**: Otro programa usa el puerto 3000.

**Soluciones**:
- Cambia el puerto en `Backend/.env`:
  ```env
  PORT=3001
  ```
- O mata el proceso en el puerto 3000:
  ```bash
  # Windows
  netstat -ano | findstr :3000
  taskkill /PID <PID> /F
  
  # macOS/Linux
  lsof -i :3000
  kill -9 <PID>
  ```

### 3. "Port 5173 is already in use"

**Causa**: Otro programa usa el puerto 5173.

**Soluciones**:
```bash
# Ejecuta en otro puerto
npm run dev -- --port 5174
```

### 4. "Cannot find module" error

**Causa**: Dependencias no instaladas correctamente.

**Soluciones**:
```bash
# En Backend/ o Frontend/, según corresponda
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```

### 5. CORS errors en la consola del navegador

**Causa**: Backend no permite solicitudes desde el frontend.

**Información**: El backend ya tiene CORS configurado, pero si ves errores:
- Verifica que el backend esté corriendo
- Revisa que la URL en `Frontend/src/plugins/pluginAxios.js` es correcta
